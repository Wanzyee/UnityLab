Detailed Description

Attribute to set execution order of a runtime UnityEngine.MonoBehaviour.

Apply if different after script assets loaded, then reload if changed. The order range from early -32000 to late 32000, 0 for default time. Set overwrite to modify the existing order with log if different. Otherwise, silently ignore attribute and keep the origin without log.

An example to make Unity to execute a custom component earlier below:

[ExecutionOrder(-100, true)]
public class SomeComp : MonoBehaviour{
private void Awake(){ Debug.Log("The early worm gets caught by the bird."); }
}
