Detailed Description

Run coroutine globally for static usage or classes not UnityEngine.MonoBehaviour.

Note, this creates hidden temporary UnityEngine.MonoBehaviour to run the coroutine. DON'T find to stop or deactive it, otherwise interrupt the coroutine silently. And this only works in play mode to be called by Unity, and avoid making the edited scene dirty.

An example below to use it easily just like UnityEngine.MonoBehaviour. Even as nested coroutine, just yield return the Coroutiner.

private void Wait(){
Coroutiner.Start(MainWait());
Debug.Log("Start main");
}
private IEnumerator MainWait(){
yield return Coroutiner.Start(NestWait());
Debug.Log("Finish main");
}
private IEnumerator NestWait(){
yield return new WaitForSeconds(1f);
Debug.Log("Finish nest");
}

And the examples to invoke repeating and delay call below:

Coroutiner.Invoke(() => Debug.Log(Time.time), 1f, 1f);
Coroutiner.Delay(() => Debug.Log("Delay Call"), new WaitForSeconds(2f));
